3.HashMap

默认的构造函数在java8中只会初始化负载因子为0.75，在第一次添加元素时调用扩容方法resize()，在此时进行初始化，初始容量为默认量16。初始化table发生在第一次put，对于无参构造，一切都是初始化的16和0。75，至于有参构造，则会在构造时初始化threshold（The next size value at which to resize (capacity * load factor).）为给定的容量向上取整的2的整数幂，接着在put时使用该容量进行初始化，最后再更新为正确的threshold。

实现原理：

hashMap类中的元素是Node类，node类除了基础的hash，key，value外，还有next，记录下一个node，因此hashMap中保存的是transient Node<K,V>[] table，记录每一个头节点，即可遍历整个map。table的大小是2的次方，方便用与运算进行散列（以及1.8后的扩容），对于散列后相同的node，将其直接添加到头节点之后，形成链表，当链表的长度大于8后，就转换为红黑树（当table的大小小于64时会先扩容table）

散列函数：

对key进行哈希计算，然后将hash值和table.length取模，此处table的长度是2的次方就可以很好的利用所有的空间。

扩容：

hashMap的扩容加载因子默认为0.75，即当hashMap中储存的Node的数量超过四分之三的table数组的长度后会触发扩容，扩容大小为两倍的原table的长度。除此之外，当table的大小小于64时，如果hash碰撞形成的链表长度大于8时，该链表并不会转换为红黑树，而是转而对table进行扩容。
